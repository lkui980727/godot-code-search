# Code Search 插件
## Code Search 是一个功能强大的 Godot 4 编辑器插件，为游戏开发者提供便捷的代码搜索、替换、预览、批量操作以及场景树导入等增强功能。支持正则表达式、大小写敏感、多文件范围、整个项目搜索，并可与编辑器无缝集成。

# ✨ 主要特性 
- ## 实时搜索与高亮：输入即搜索，匹配行以自定义颜色高亮，支持多行文本、正则表达式、大小写切换。
- ## 智能搜索范围：当前文件、所有打开文件、整个项目。
- ## 替换功能：全部替换（支持当前文件/所有打开文件/整个项目），替换前确认对话框。
- ## 结果导航：树形分组显示结果，支持上一个/下一个导航、批量跳转。
- ## 右键菜单：复制行内容、复制文件路径、在文件夹中显示、预览上下文。
- ## 批量操作：批量复制文件路径、批量跳转。
- ## 导出搜索结果：将匹配结果导出为文本文件，包含文件路径和行范围。
- ## 搜索历史：自动记录搜索词，按时间排序，可清空。
- ## 国际化：内置中英文切换，界面文本实时更新。
- ## 编辑器集成：工具栏菜单按钮、可停靠面板、浮动弹窗、快捷键（Ctrl+Shift+F 弹窗，Ctrl+Shift+D 面板，支持自定义）。
- ## 主题适配：自动跟随 Godot 编辑器主题。
- ## 导入场景树：支持 JSON、树形文本（带 ├─ 符号）、Godot 场景文件（.tscn）三种格式，快速生成场景节点树。
- ## 文件类型过滤：搜索时可选择仅脚本（.gd）、包含资源文件（.tscn, .tres, .json, .txt, .cfg）或所有文本文件。*

# 📦 安装 
1. ### 下载插件压缩包，解压后得到 addons/code\_search/ 文件夹。
1. ### 将整个 addons 文件夹复制到您的 Godot 项目根目录。
1. ### 打开 Godot 编辑器，点击菜单 项目 → 项目设置 → 插件。
1. ### 找到 Code Search 插件，勾选 启用 即可。
- ### 注意：插件需要 Godot 4.0 或更高版本，推荐使用 4.2+。

# 🚀 使用指南 
## 基本操作 
- ### 打开停靠面板：点击工具栏的 “代码搜索” 按钮，选择 “停靠面板”（或按快捷键 Ctrl+Shift+D）。

- ### 打开浮动弹窗：点击工具栏的 “代码搜索” 按钮，选择 “浮动弹窗”（或按快捷键 Ctrl+Shift+F）。

- ### 搜索：在搜索框中输入关键词，匹配行将自动高亮。支持多行输入，直接粘贴代码片段。

- ### 切换选项：点击 Aa 按钮切换大小写敏感，点击 .\* 按钮启用正则模式。

- ### 选择范围：使用下拉框选择搜索范围（当前文件/所有打开文件/整个项目）。

- ### 选择文件类型：在“文件类型”下拉框中限制搜索的文件扩展名。

- ### 替换：在替换框中输入替换文本，点击“全部替换”按钮，确认后执行替换。

- ### 结果导航：点击“上一个/下一个”按钮，或直接在结果树中点击匹配项，即可跳转到对应位置。

- ### 多选：按住 Ctrl 或 Shift 键可多选结果项，然后使用“批量跳转”或“批量复制路径”。

- ### 右键菜单：在结果项上右键，可复制行内容、复制文件路径、在文件管理器中显示、预览代码片段。

- ### 导出结果：点击“导出”按钮，选择保存路径，将当前搜索结果导出为 .txt 文件。

- ### 清空搜索：点击“清空”按钮，清空搜索框和替换框内容，并清除高亮。

- ### 历史记录：点击“历史”按钮，可查看最近使用的搜索词，点击即可重用。点击“清空历史”可清除记录。

# ⚙️ 配置与快捷键
## 自定义快捷键 
1. ### 点击工具栏菜单按钮，选择 “设置”。

2. ### 在弹出的对话框中，输入您希望的字母键（A-Z），然后点击对应按钮（“设置弹窗快捷键”或“设置面板快捷键”）。

3. ### 新快捷键（Ctrl+Shift+字母）将立即生效并保存。

## 修改界面语言 
- ### 点击工具栏菜单按钮，选择 “语言” → “English” 或 “中文”，界面文本会实时切换。

# 🛠️ 开发与扩展
 
## 自定义导入器 
### 若需要支持更多剪贴板格式（如 XML、YAML），可在 addons/code\_search/importers/ 目录下添加脚本，实现 can\_parse(text) 和 parse(text) 静态方法。返回的可以是 Dictionary（JSON 格式）或 Node。插件会自动调用。

## 扩展属性支持
### 在 indent\_importer.gd 的 \_apply\_node\_properties 函数中，可增加对其他节点类型和属性的解析逻辑。

# ❓ 常见问题
## Q：搜索整个项目时，点击“上一个/下一个”总是弹出对话框？ 
### A：已修复。插件现在只对 .gd 文件调用打开对话框，其他文件类型会跳过并输出警告。

## Q：替换整个项目无效？ 
### A：请检查 Output 面板是否有“无法写入文件”的提示。可能是文件权限问题，或扩展名过滤未包含目标文件。默认已包含 .gd、.tscn 等常见文本文件。

## Q：导入树形文本后，只有根节点没有子节点？ 
### A：请确保文本中的缩进和树形符号（├─、└─）正确，每行格式为 类型: 名称 或 名称 (实际类型)。若节点类型不存在，会输出警告。

## Q：高亮颜色改变后，匹配行没有更新？ 
### A：这是正常行为。改变颜色后，插件会重新搜索，并使用新颜色高亮。若未自动搜索，可手动点击“清空”后再输入。

## Q：插件与我的其他插件冲突吗？ 
### A：插件使用独立的命名空间和编辑器接口，与其他插件无直接冲突。若遇到快捷键冲突，可在“设置”中修改快捷键。
